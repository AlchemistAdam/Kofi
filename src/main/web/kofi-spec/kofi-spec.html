  1. KoFi Text Syntax

1. Introduction

The KoFi text syntax is a collection of syntactic rules for representing Java arrays, objects and primitive values in a human-readable format.

All rules are written in a highlighted block:

lf = %A ;

Later sections frequently use Unicode characters. These are written on a highlighted background with the character first, if visible, followed by the character name: % Percent Sign.


2. Notation

Rules

Syntactic rules are written in Extended Backus-Naur Form notation. Each rule is composed of a symbol, an assignment operator, an expression and a termination operator:

symbol = expression ;

Expressions are evaluated left-to-right.


Hexadecimal values corresponding to Unicode codepoints are used for characters that cannot be represented as a single visible character, starting with a % Percent Sign:

lineFeed = %A ;

defines a symbol that matches a single Line Feed character.

For wildcard characters, a . Full Stop is used:

anyCharacter = . ;

Character ranges match a subset of Unicode characters. The beginning and ending characters are surrounded with ' Apostrophe characters and separated with a _ Low Line:

A_Z = 'A'_'Z' ;

defines a symbol that matches all characters between "A" and "Z" (inclusive).


All other values are surrounded with " Quotation Mark or ' Apostrophe characters and are case-insensitive:

A = "A" ;

defines a symbol that matches both "A" and "a".

Group Operators

Expressions can be either a single terminal value, or a group of terminal values and other symbols. Values and symbols can be combined with group operators, and are listed from highest to lowest precedence:

  • Concatenation — X and Y.

    X , Y

  • Exception — X, but not Y.

    X - Y

  • Alternation — X or Y.

    X | Y

Bracket Operators

The pattern of values and groups can be altered with bracket operators:

  • Optional — X or nothing.

    [X]

  • Optional Repetition — X one or more times, or nothing.

    {X}

  • Repetition — X one or more times.

    {X}+

Precedence

The evaluation order of values and groups is changed by surrounding them with parentheses. Parenthesized values and groups have higher precedence and are evaluated first, starting at the deepest level.


3. Document

Text that conforms to the KoFi text syntax is referred to as a document; a sequence of zero or more elements, each separated with a Line Feed character. Elements can have both prefixed and suffixed whitespace. A document is defined as:

document   = [docElement , {lf , docElement}] ;
docElement = [ws] , element , [ws] ;
lf         = %A ;
ws         = {%9 | %D | %20}+ ;

The "nc" symbol defines a valid character and is used by other symbols. It allows for a character to be escaped, and is defined as:

nc = . - (lf | "\") | "\" , (. - lf) ;

4. Elements

There are four types of elements:

element = whitespace | comment | section | property ;

Whitespace elements do not carry any information. They are empty strings or strings consisting of only whitespace characters, and separate other elements to increase readability:

whitespace = [ws] ;

Comment elements describe other elements and the document itself. A comment begins with a ; Semicolon followed by a possibly empty string:

comment = ";" , {. - lf} ;

Section elements group other elements together. A section begins with a [ Left Square Bracket, followed by a possibly empty section name, and ends with a ] Right Square Bracket:

section = "[" , {nc} , "]" ;

Property elements store data as a key-value pair. The value of a property is complex and can be many things; it is covered in its own section. A property begins with a possibly empty property name, followed by an = Equals Sign, and ends with a property value:

property     = [propertyName] , [ws] , "=" , [ws] , value ;
propertyName = pns , [{pnc} , pne] ;
pns          = pne - (";" | "[") ;
pne          = pnc - ws ;
pnc          = nc - "=" ;

; Semicolon and [ Left Square Bracket cannot be used as the first character of a property name, unless escaped, as these are used to begin comment and section elements. Nor can the name contain the = Equals Sign, unless escaped, as it is used to separate the name from the value.


5. Values

Property values can be null, primitive Java values, strings, arrays or objects:

value = null | boolean | string | char | number | array | object ;

Null and boolean values are the simplest, and are self-explanatory:

null    = "null" ;
boolean = "true" | "false" ;

String values are surrounded with " Quotation Mark characters. " Quotation Mark characters inside the string must be escaped:

string = '"' , {nc - '"'} , '"' ;

Character values are surrounded with ' Apostrophe characters. ' Apostrophe and \ Reverse Solidus characters must be escaped. They can be a single character, a two-character escape sequence or a six-character escape sequence:

char     = "'" , (. - (lf | "\" | "'") | 2esc | 6esc) , "'" ;
2esc     = "\0" | "\b" | "\t" | "\n" | "\f" | "\r" | "\\" | "\'" | '\"' ;
6esc     = "\u" , hexDigit , hexDigit , hexDigit , hexDigit ;
hexDigit = '0'_'9' | 'A'_'F' | 'a'_'f' ;

Number values include all primitive number types in Java. Numbers can be a sequence of digits, or an infinity or NaN constant. Type specifiers can be appended to denote value types of long, float and double, and floating-point numbers can have an exponent:

number    = [sign] , (integer | float | "infinity") | "nan" ;
integer   = int , ["L" | "F" | "d"] ;
float     = fraction , [exponent] , ["F" | "d"] ;
fraction  = [int] , "." , [int] - "." ;
exponent  = "e" , [sign] , int ;
sign      = "+" | "-" ;
int       = {'0'_'9'}+ ;

Array values are an indexed sequence of property values, separated with a , Comma. An array begins with a [ Left Square Bracket and ends with a ] Right Square Bracket:

array   = "[" , [entity , {"," , entity}] , "]" ;
entity  = [ws] , value , [ws] ;

Object values are an entry-based map of property values, separated with a , Comma. An object begins with a { Left Curly Bracket and ends with a } Right Curly Bracket:

object = "{" , [entry , {"," , entry}] , "}" ;
entry  = [ws] , string , [ws] , ":" , value , [ws] ;
